# Jarvis Telegram-бот (Фаза 2 — текстовый вход)

Лёгкий long-polling бот поверх локального rag-cms. Принимает текст → спрашивает
агента памяти → возвращает ответ с цитатами. Команда «запомни X» дописывает факт
в память и реиндексит.

## Файлы
- `jarvis_api.py` — клиент к rag-cms (логин/JWT, запрос агента, запись памяти)
- `jarvis_bot.py` — Telegram long-polling + обработчики
- `.env` — конфиг (gitignored; RAG_ID/логин уже вписаны, нужен токен бота)
- `notes/` — локальные копии сохранённых через «запомни» фактов

## Настройка
1. Создать бота: написать [@BotFather](https://t.me/BotFather) → `/newbot` →
   скопировать токен вида `123456789:ABCdef...`
2. Вписать в `.env`: `TELEGRAM_BOT_TOKEN=<токен>`
3. (Рекомендуется) узнать свой Telegram ID — написать боту `/start`, он покажет
   ID → вписать в `.env`: `JARVIS_ALLOWED_USERS=<твой_id>` (иначе бот отвечает всем).

## Запуск
```bash
pip install -r requirements.txt   # только requests
python jarvis_bot.py
```
Backend rag-cms должен быть поднят (`docker compose ... up -d`), см. `../plan.md`.

## Команды
- любой текст → **лёгкий путь**: ретрив + 1 LLM-вызов (~5-8с, ~3K токенов).
  Быстро и дёшево, отвечает по источникам. Лучший выбор для точечных фактов.
- `/deep <вопрос>` → **тяжёлый ReAct-агент**: многошаговое рассуждение
  (~30-50с, ~45K токенов). Для сводных/сложных вопросов («перечисли все задачи»).
- `/task <описание>` (алиасы `/задача`, `/сделай`) → **билдер-агент по коду**:
  заводит ветку, делает минимальные правки, гоняет тесты и присылает отчёт.
- `запомни <факт>` → сохранить новый факт в память
- `/start`, `/help` → справка + показать твой Telegram ID

## Два пути ответа
| | Лёгкий (по умолчанию) | `/deep` (агент) |
|---|---|---|
| Вызовов LLM | 1 | 7-10 |
| Токенов | ~3K | ~45K |
| Время | 5-8с | 30-50с |
| Сила | точные факты | многошаговые/сводные |

Лёгкий путь делает прямой Groq-вызов с **ротацией по 3 ключам** (`GROQ_API_KEYS`)
— при 429 на одном ключе автоматически переходит к следующему. ~100 вопросов/день
на $0 против ~6 у чистого агента.

## Ограничения (MVP)
- Синхронный: один запрос за раз. Для личного использования ок.
- «запомни» создаёт отдельный .md на факт и реиндексит весь RAG (~30с).
- Бот живёт, пока запущен `python jarvis_bot.py` (24/7 хостинг — Фаза 3).
